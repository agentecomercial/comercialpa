🔑 Configuração inicial — OAuth Google

Pra ler agendas privadas (compartilhadas com sua conta), o Google exige login OAuth. Setup é uma vez só.

  1. Abra console.cloud.google.com e selecione o projeto (ou crie um novo)
  2. Habilite a Google Calendar API: ☰ → APIs e serviços → Biblioteca → busca por Google Calendar API → Ativar (se já fez antes, pula)
  3. Configure a tela de consentimento OAuth: ☰ → APIs e serviços → Tela de consentimento OAuth:
    • Tipo: Interno (só @febracis.com.br — sem aprovação Google necessária)
    • Nome do app: Agenda Febracis
    • Email de suporte: seu email
    • Salvar e voltar (pode pular "Escopos" e "Resumo")
  4. Crie o Client ID OAuth: ☰ → APIs e serviços → Credenciais → + Criar credenciais → ID do cliente OAuth
    • Tipo: Aplicativo da Web
    • Nome: Agenda Febracis Web
    • Em Origens JavaScript autorizadas, adicione 2 entradas:
      http://localhost:5500
      http://127.0.0.1:5500
    • Deixe "URIs de redirecionamento autorizados" em branco
    • Criar → copie o ID do cliente (termina em .apps.googleusercontent.com)
