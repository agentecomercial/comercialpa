📚 Manual do Inteligência Comercial

Este manual documenta o módulo Inteligência Comercial (IC) embutido no dashboard. Cobre o que cada aba entrega, como os scores são calculados, onde os dados moram e como restaurar o sistema em caso de falha. Acesso restrito a ADM e ao usuário EXTRACLASSE.

Para quem é este manual:
· Gestores que precisam diagnosticar comportamento do time
· Quem vai dar manutenção no código (você ou outro dev)
· Suporte tier 2 quando algo "não aparece" ou "não bate"

🏗 Arquitetura

O IC vive integralmente dentro de #mapeamentoScreen no dashboard.html. Cada aba é um .ic-pane[data-icpane] e os módulos JS associados ficam em assets/js/40-*.js a 46-*.js.

ArquivoFunção
30-sync-ferramentas.jsCarrega _mapDados (turmas → registros consolidados)
40-ic-feedback.jsFeedback + 9 competências + drill-down
41-ic-pdi.jsPDI trimestral + chips sugeridos
42-ic-ia.jsIA híbrida (catálogo + copy-paste)
43-ic-alertas.jsAlertas estratégicos + Pareto
44-ic-playbook.jsBiblioteca de scripts (147 seeds)
45-ic-cobranca.jsPromessas de pagamento
46-ic-manual.jsEste manual (mostra/esconde aba conforme perfil)

Hooks principais: _icShowPane(name) troca de aba. Cada módulo escuta esse hook para inicializar sua própria carga.

Fase 1 · 📊 Dashboard

Aba principal. Mostra KPIs cross-turmas: faturamento total, clientes pagos, consultores ativos, treinamentos, maior performance. Filtros de Ano e Meses no topo.

Origem dos dados: _mapCarregar() em 30-sync-ferramentas.js lê turmas/* do Firebase, achata em registros (granular por sub-treinamento pago), guarda em _mapDados.

Regra crítica: só sub-treinamentos com status==='pago' entram. Status global do cliente é ignorado quando há array treinamentos.

Fase 2 · 📈 Feedback (Desenvolvimento)

Avaliação por consultor em 9 competências. Scores autocalculados a partir dos dados do mês. Drill-down ao clicar em cada competência mostra os clientes que compuseram a nota.

Janela de análise: default = só mês do ciclo. Pode expandir até 24 meses ou "tudo".

Schema dos clientes: turmas usam cliente/consultor/treinamento. Vendas avulsas usam clienteNome/consultorNome/produto. _icFbAchatar normaliza os dois.

Modos auto/manual: se o gestor mexer no gráfico → vira "manual". Sem alteração → "automatico".

Fase 3 · Competências (detalhe)

Veja a seção 9 competências abaixo para fórmulas.

Fase 4 · 🌱 PDI

Plano de Desenvolvimento Individual com períodos flexíveis (mês inicial + mês final + chips de atalho). Cada consultor pode ter 1 PDI ativo por período.

  • Diagnóstico: baseado nos últimos 3 ciclos de feedback do consultor.
  • Competências-alvo: até 3 por PDI. Chip ⭐ sugere as 3 mais baixas automaticamente.
  • Estado: iniciado / concluido.
  • Persistência: icPDIs/{consultor}/{trimestreId}.

Fase 5 · 🤖 IA (híbrida)

Sistema sem API. (A) Catálogo de análises estruturadas geradas localmente a partir dos dados em memória. (B) Pergunta livre que monta prompt completo (system + snapshot + pergunta) e copia para o clipboard — o usuário cola no Claude.ai (conta MAX, browser).

  • Período: mês corrente · 3m · 6m · manual.
  • Dados enviados: turmas + pipelineSales + pipelineGoals + últimos 3 ciclos de feedback + PDIs ativos.
  • Histórico: icIAHist/{ts}.
  • Restrição: aba aparece só pra ADM/EXTRACLASSE.

Fase 6 · 🚨 Alertas estratégicos + Pareto

Banner de alertas no topo do Dashboard + gráfico Pareto de concentração de faturamento.

6 regras de alerta:

  1. Dependência crítica: top 1 ≥ 50% do faturamento.
  2. Dependência alta: top 1 ≥ 35%.
  3. Inatividade: consultores sem nenhum pago no período.
  4. Produto parado: 2+ treinamentos sem venda.
  5. Feedback atrasado: consultor sem ciclo há mais de 14 dias.
  6. PDI faltando: score < 7 sem PDI ativo.
  7. Pareto 80/20 desbalanceado: ≤30% dos consultores fazem 80% do faturamento.

Hook: MutationObserver em #mapKpis — recalcula automaticamente sempre que o render do mapeamento muda.

Preview standalone: ic-alertas-pareto-preview.html na raiz tem 4 cenários (concentrado/saudável/crítico/vazio).

Fase 7 · 📖 Playbook

Biblioteca de scripts de venda. 147 scripts sincronizados a cada bootstrap:

  • 19 genéricos (objeções clássicas como "tá caro" / "vou pensar", fechamento, follow-up, cobrança).
  • 128 específicos por produto (16 produtos × 8 categorias: abertura, qualificação, apresentação, objeções, fechamento, follow-up, recuperação, cobrança).

IDs determinísticos (pb_seed_<slug>): bootstrap é idempotente — clicar de novo sobrescreve sem duplicar. Botão 🧹 Limpar seeds remove só os seeds, preservando scripts criados manualmente.

Persistência: icPlaybook/{id}. Cada script tem categoria, título, situação, texto, produtos[], tags[].

Fase 8 · 💰 Cobrança Saudável

Acompanhamento de promessas de pagamento. Cada consultor registra à mão; sistema acompanha vencimento e mede % honradas.

EstadoQuandoConta no KPI?
🟡 PendentePromessa em aberto, ainda no prazoNão
🔴 VencidaPassou da data, sem ação ainda (auto)Não
✅ HonradaCliente pagou+1 honrada
❌ QuebradaCliente não pagou e não vai pagar+1 quebrada (puxa % pra baixo)
↻ ReagendadaNova data prometida — cria nova promessa vinculadaNeutra

Integração com Playbook: botão "📋 Script" em cada card busca o script de cobrança correspondente ao estado e copia para o clipboard.

Persistência: icCobrancas/{consultor}/{id}.

🎯 9 Competências (fórmulas)

Cada competência gera um score 0–10 a partir dos clientes do período. O drill-down ao clicar mostra exatamente quais clientes entraram na fórmula.

CompetênciaFórmulaCliente avaliado
Prospecçãopagos + neg + ent ÷ totalTodos clientes do período
Qualificação(pagos+neg+ent) ÷ (pagos+neg+ent+aberto+desistiu)Inclui "—" e "DESISTIU" como aberto
ApresentaçãoManual (gestor)—
Negociaçãopagos ÷ (pagos+neg+ent)Filtros Turma/Avulsa toggle
Follow-upConversão entre etapas—
ConstânciaDesvio do ritmo mensal—
MixDiversidade de produtos vendidos—
Aproveitamentopagos ÷ (turma + avulsas)2 cards: 🏫 Turma · 🛒 Avulsa
Visão (combinada)Preservação 40% + Potencial 40% + Recuperação 20%Janela móvel de turmas

Conversão final: exibida ao lado da Negociação, complementar à taxa de perda. Não afeta o score.

❓ FAQ / Troubleshooting

Q: O painel Pareto não aparece.
R: Verifique se há ao menos 1 cliente pago no período filtrado. Se ainda assim não aparecer, confira no console se window._mapDados está populado. O painel só renderiza com totFat > 0.
Q: Bootstrap do Playbook duplicou scripts.
R: A versão antiga usava IDs aleatórios. Clique em 🧹 Limpar seeds (remove só os pb_seed_*) e depois ⚡ Bootstrap inicial de novo — agora com IDs determinísticos é idempotente.
Q: Vendas avulsas não aparecem nas competências.
R: Garanta que os registros têm clienteNome / consultorNome / produto (schema avulsas). O _icFbAchatar faz fallback para cliente / consultor / treinamento, mas exige que ao menos um par esteja preenchido.
Q: Atualizei o código e não vejo a mudança no navegador.
R: Rode bump.cmd (ou scripts/bump-cache.ps1) — incrementa o ?v=20260603-151115"mn-faq"> Q: Promessa de cobrança ficou "Pendente" mesmo após a data.
R: O estado Vencida é calculado em tempo de render (não persistido). Se a promessa está marcada manualmente como Honrada/Quebrada/Reagendada, esses tem prioridade. Caso contrário, basta abrir a aba que ela aparece como Vencida.
Q: Como dar acesso de admin para alguém?
R: Use o usuário EXTRACLASSE (já tem acesso) ou defina perfil: 'adm' no nó usuarios/<login> do Firebase.
Q: A IA pediu pra "copiar e colar no Claude" — onde colo?
R: Abra claude.ai em outra aba (use sua conta MAX), cole o prompt completo. O Claude responde, você volta no dashboard e cola a resposta em "Cole a resposta do Claude aqui" para salvar no histórico.

📖 Glossário

Sub-treinamento
Linha do array treinamentos[] dentro de um cliente. Tem código, valor e status próprios.
Ciclo de feedback
Avaliação pontual das 9 competências de um consultor em uma data específica. Pode ter modo auto ou manual.
PDI
Plano de Desenvolvimento Individual: até 3 competências-alvo + ações em um período.
Janela de análise
Quantos meses para trás o feedback considera ao calcular scores. Default: 0 (só mês do ciclo).
Pareto / 80-20
Quantos consultores compõem 80% do faturamento — indica concentração de risco.
Promessa honrada
Cliente pagou conforme combinado. Compõe % honradas.
Promessa quebrada
Cliente não pagou e não vai mais pagar. Sinal de abordagem que precisa ajuste.
Seed
Script do Playbook criado pelo bootstrap (ID começa com pb_seed_). Pode ser apagado em massa.
